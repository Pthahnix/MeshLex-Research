## 数据集 Pipeline 详细说明 (Phase D)

### 概览

Phase D 的目标是将 Objaverse-LVIS 和 ShapeNetCore v2 两大 3D 数据集统一处理为 patch-level Parquet 数据集，上传到 HuggingFace，供后续训练使用。

### HF 数据集: `Pthahnix/MeshLex-Patches`

**格式**: Apache Parquet (通过 Daft DataFrame 引擎写入)

#### 数据来源

| 来源 | 成功 Meshes | 失败 Meshes | 成功率 | Patches 总数 | Avg Patches/Mesh |
|------|------------|------------|--------|-------------|-----------------|
| Objaverse-LVIS | 32,136 | 14,364 | 69.1% | 4,619,061 | 143.7 |
| ShapeNetCore v2 | 40,419 | 12,053 | 77.0% | 6,186,952 | 153.1 |
| **合计** | **72,555** | **26,417** | **73.3%** | **10,806,013** | **148.9** |

#### Parquet Schema (每行 = 一个 patch)

| 列名 | 类型 | 说明 |
|------|------|------|
| `mesh_id` | string | Mesh 唯一 ID（Objaverse: uid, ShapeNet: synset_modelId） |
| `patch_idx` | int | 该 mesh 内 patch 的编号 |
| `category` | string | 类别名（Objaverse: LVIS 类别, ShapeNet: synset 名） |
| `source` | string | `"objaverse"` 或 `"shapenet"` |
| `n_faces` | int | 该 patch 的面数 |
| `n_verts` | int | 该 patch 的顶点数 |
| `faces` | binary (flattened int32) | 面索引数组 (n_faces × 3) |
| `vertices` | binary (flattened float32) | 全局坐标顶点 (n_verts × 3) |
| `local_vertices` | binary (flattened float32) | **PCA 归一化后的局部坐标** (n_verts × 3) |
| `local_vertices_nopca` | binary (flattened float32) | **无 PCA 的局部坐标**（仅平移+缩放） |
| `centroid` | binary (float32 × 3) | Patch 质心（全局坐标） |
| `principal_axes` | binary (float32 × 9) | PCA 主轴 (3×3 旋转矩阵) |
| `scale` | float | 归一化缩放因子 |
| `boundary_vertices` | binary (int32) | 边界顶点索引 |
| `global_face_indices` | binary (int32) | 在原始 mesh 中的面索引 |

#### Dual Normalization 设计

每个 patch 同时存储两种归一化形式：
1. **`local_vertices` (PCA)**: 先平移到质心原点 → PCA 旋转对齐主轴 → 缩放到 [-1,1]
2. **`local_vertices_nopca`**: 仅平移到质心原点 → 缩放到 [-1,1]（保留原始朝向）

这样 Phase B（PCA + Rotation Tokens）和 Phase C（No-PCA baseline）可以直接从同一数据集读取各自需要的表示，无需重新处理。

### 后续如何使用

#### 1. 加载数据集 (Python)

```python
import daft
from src.daft_utils import get_hf_io_config

io_config = get_hf_io_config()
df = daft.read_parquet(
    "hf://datasets/Pthahnix/MeshLex-Patches/**/*.parquet",
    io_config=io_config,
)

# 按 source 过滤
objaverse_df = df.where(df["source"] == "objaverse")
shapenet_df = df.where(df["source"] == "shapenet")

# 按 category 过滤
chairs = df.where(df["category"] == "chair")
```

#### 2. 加载 Splits

```python
import json
from huggingface_hub import hf_hub_download

splits_path = hf_hub_download(
    repo_id="Pthahnix/MeshLex-Patches",
    filename="splits.json",
    repo_type="dataset",
)
with open(splits_path) as f:
    splits = json.load(f)

train_ids = set(splits["seen_train"])
test_ids = set(splits["seen_test"])

# 过滤 Daft DataFrame
train_df = df.where(df["mesh_id"].is_in(train_ids))
```

#### 3. 解码 Patch 数据 (binary → numpy)

```python
import numpy as np

row = df.limit(1).collect().to_pydict()
verts_bytes = row["local_vertices"][0]
n_verts = row["n_verts"][0]
vertices = np.frombuffer(verts_bytes, dtype=np.float32).reshape(n_verts, 3)
```

#### 4. Phase B/C 训练流程

Phase B (PCA + Rotation Tokens):
- 读取 `local_vertices` 列作为 patch 输入
- 读取 `principal_axes` + `centroid` + `scale` 作为 assembly token
- 训练 RVQ VQ-VAE → 编码为 token sequence → 训练 AR

Phase C (No-PCA baseline):
- 读取 `local_vertices_nopca` 列作为 patch 输入
- 只需 `centroid` + `scale`（无需 rotation）
- 同样的 RVQ → AR pipeline

### Pipeline 技术细节

#### 处理流程

```
原始 mesh (GLB/OBJ)
  → pyfqmr 降面到 4000 faces
  → 归一化到 [-1,1]
  → METIS 图分割 (target_patch_faces=35)
  → 每个 patch:
      - 提取局部顶点和面
      - PCA 归一化 → local_vertices
      - 无 PCA 归一化 → local_vertices_nopca
      - 记录 centroid, principal_axes, scale, boundary_vertices
  → Daft DataFrame → Parquet → HF upload
```

#### 关键参数

| 参数 | 值 | 说明 |
|------|-----|------|
| `target_faces` | 4000 | 降面目标 |
| `target_patch_faces` | 35 | METIS 分割目标面数/patch |
| `sub_batch_size` | 2000 (ShapeNet) / 500 (Objaverse) | 每次 HF commit 的 mesh 数 |
| `max_faces` | 128 | 单个 patch 最大面数 |
| `max_verts` | 128 | 单个 patch 最大顶点数 |

#### Resume 机制

Pipeline 通过 `progress.json` 追踪已完成的 batch/category：
- Objaverse: 按 500 个 UID 为一个 batch
- ShapeNet: 按 synset category 为单位
- 重启时自动跳过已完成的单元
- HF upload 在每个 sub-batch 后执行，确保不丢数据

#### 已知问题

1. **ShapeNet 2 个类别不可用**: bicycle (02834778) 和 boat (02858304) 在 HF 返回 404
2. **HF 速率限制**: 128 commits/hour，大类别需要用 sub_batch_size=2000 减少 commit 次数
3. **Vehicle 类别成功率低**: car 14%, motorbike 23%, train 34%（复杂车辆 mesh 降面后容易失败）
4. **Parquet 行数 vs metadata 行数有微小差异**: Parquet 11,479,235 rows vs metadata 报告 10,806,013 patches（部分 mesh 有 patches 写入但元数据缺失）
