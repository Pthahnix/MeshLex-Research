# TRELLIS: Structured 3D Latents for Scalable and Versatile 3D Generation

## Core Architecture and Representation

TRELLIS introduces **Structured Latents (SLat)**, a unified 3D latent representation built on a sparsely-populated 3D grid. Each 3D asset is encoded as a set of local latent vectors attached to active voxels that intersect the object's surface:

- **Representation**: z = {(z_i, p_i)} where p_i are active voxel positions in a 64^3 grid and z_i are local latent vectors (8 channels).
- **Sparsity**: Average ~20K active voxels out of 64^3 = 262K total, enabling efficient high-resolution modeling.
- **Encoding**: Dense multiview images are rendered from the 3D asset, features extracted via pretrained DINOv2 (ViT), projected onto active voxels, and averaged. A transformer-based Sparse VAE then compresses these into compact latents.
- **Versatile decoding**: Three separate decoders map SLat to different output formats:
  - 3D Gaussians (32 per voxel, with position offsets, colors, scales, opacities, rotations)
  - Radiance Fields (CP-decomposed local 8^3 volumes via Strivec)
  - Meshes (FlexiCubes at 256^3 resolution with SDF values)
- **VAE architecture**: Transformer with 3D shifted window attention (8^3 windows) for local information interaction. Encoder trained end-to-end with Gaussian decoder; other decoders trained separately with frozen encoder.

## Generation Pipeline

Two-stage generation using **rectified flow transformers**:

1. **Stage 1 - Sparse Structure Generation**: Binary 3D occupancy grid compressed via a 3D convolutional VAE (64^3 to 16^3), then generated by a transformer with adaLN timestep conditioning and cross-attention for text/image conditions.
2. **Stage 2 - Structured Latent Generation**: Given the sparse structure, latents are generated by a second transformer using sparse convolution downsampling/upsampling blocks. Latents are packed into shorter sequences via 2^3 local pooling for efficiency.

Conditions injected via cross-attention: CLIP features for text, DINOv2 features for images. Classifier-free guidance with 10% drop rate.

## Model Scale and Training Data

- **Three model sizes**: Basic (342M), Large (1.1B), X-Large (2B total parameters).
- **Training data**: ~500K curated 3D assets from Objaverse (XL), ABO, 3D-FUTURE, and HSSD. 150 images rendered per asset. GPT-4o used for captioning.
- **Training**: XL model trained on 64 A100 GPUs (40G) for 400K steps, batch size 256. AdamW optimizer, lr=1e-4.
- **Inference**: CFG strength 3, 50 sampling steps.

## Conditioning Mechanism

- **Text conditioning**: CLIP text features injected via cross-attention in both generation stages.
- **Image conditioning**: DINOv2 visual features injected via cross-attention.
- **Data augmentation**: Text prompts summarized to varying lengths; images rendered with different FoVs.
- **Classifier-free guidance**: 10% condition drop rate during training.

## Key Innovation

TRELLIS unifies 3D generation across multiple output representations (Gaussians, Radiance Fields, meshes) through a single sparse voxel-based latent space enriched with DINOv2 visual features, eliminating the need for costly per-asset 3D fitting while enabling format-agnostic generation and tuning-free local editing via a two-stage rectified flow pipeline.

## Main Limitations

- **Object-level only**: Cannot generate large-scale scenes; limited to individual or small groups of objects.
- **Two-stage inference**: Sequential structure-then-latent generation adds pipeline complexity and potential error accumulation.
- **Sparse structure bottleneck**: The 64^3 grid resolution, while efficient, may limit extremely fine geometric detail at the voxel level.
- **DINOv2 dependency**: Encoding quality is bounded by the vision foundation model's representation capacity; no dedicated 3D encoder is used.
- **Computational cost**: The 2B parameter XL model requires significant GPU resources for both training (64 A100s) and inference.

## Quantitative Highlights

- Reconstruction: PSNR 32.74, LPIPS 0.025 (appearance); CD 0.0083, F-score 0.9999 (geometry) — surpassing all baselines including geometry-focused CLAY.
- Generation: Significant improvements over all baselines on Toys4k across FD and KD metrics with both Inception and DINOv2 features.
- Ablations confirm: (1) 64^3 resolution outperforms 32^3 with more channels, (2) rectified flow beats diffusion at both stages, (3) scaling from B to XL consistently improves quality.
